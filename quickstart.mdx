<Warning>
This quickstart targets the production-shape API. The first cohort of integrators currently runs against a pre-production sandbox provisioned manually — [contact sales](https://fibo.io/contact) for credentials. Self-serve key minting via the dashboard ships at the **Sandbox + CLI admin** milestone.
</Warning>

## 1. Get your API keys

Two key types are issued per environment (`test` and `live`):

- **`pk_<env>_…`** — *publishable* key. Embedded in the browser bundle, safe to expose. Used by the SDK to authenticate the user via Privy and to call read-only endpoints.
- **`sk_<env>_…`** — *secret* key. Server-side only. Used by your backend to call privileged endpoints (refunds, user listing, webhook secret rotation).

```bash
# Example — your actual keys arrive over a secure channel during onboarding
FIBO_PUBLISHABLE_KEY=pk_test_xxx
FIBO_SECRET_KEY=sk_test_xxx
```

<Note>
**Never** ship `sk_…` to the browser. The SDK never asks for it. If you spot one in a frontend bundle, rotate it immediately via [`POST /v1/api-keys/{id}/rotate`](/api-reference/introduction#rotating-keys).
</Note>

## 2. Install the SDK

```bash
npm install @fibo/react-sdk
```

The SDK has Privy as a transitive dependency — you don't need to install or configure Privy yourself ([ADR 0003](https://github.com/vigimani/fibo-saas/blob/main/docs/adr/0003-privy-as-transitive-dep-in-sdk.md)).

## 3. Wrap your app

```tsx
import { FiboProvider } from '@fibo/react-sdk';

export default function App() {
  return (
    <FiboProvider
      publishableKey={process.env.NEXT_PUBLIC_FIBO_PUBLISHABLE_KEY!}
      // Optional: pass your own user identifier so Fibo can join its
      // record with your DB on the wallet.created webhook.
      externalUserId={currentUser.id}
    >
      <YourApp />
    </FiboProvider>
  );
}
```

## 4. Trigger your first action

    ```tsx
    import { useFiboWallet } from '@fibo/react-sdk';

    function WalletBadge() {
      const { addresses, isLoading } = useFiboWallet();
      if (isLoading) return null;
      return <code>{addresses.evm}</code>;
    ```

    ```tsx
    import { useFiboSwap } from '@fibo/react-sdk';

    function SwapWidget() {
      const { quote, swap } = useFiboSwap();
      // see /api-reference/introduction for the full payload shape
      return <button onClick={() => swap(quote)}>Swap</button>;
    }
    ```
  </Tab>
  <Tab title="Listen for events (server)">
    ```ts
    // Express handler — see /authentication/webhook-signatures
    app.post('/fibo/webhooks', verifyFiboSignature, (req, res) => {
      switch (req.body.event_type) {
        case 'wallet.created':  /* persist mapping */ break;
        case 'tx.confirmed':    /* update user UI / accounting */ break;
        case 'wallet.funded':   /* unlock features */ break;
      }
      res.sendStatus(204);
    });
    ```
  </Tab>
</Tabs>

## What you have now

After completing the steps above:

1. Your users authenticate via the FiboProvider (email / OAuth / passkey, configurable).
2. On first login, Fibo provisions embedded wallets on the chains you opted into.
3. You receive a `wallet.created` webhook with the wallet addresses — store these against your user record.
4. The SDK exposes hooks for balances, swaps, transfers, and yield operations. Every state-changing action returns a Promise that resolves once the user has signed.
