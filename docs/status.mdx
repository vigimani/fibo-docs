Fibo is in active development. This page is the honest reference for what's actually wired today vs. what the rest of the docs is *describing* but not yet *delivering*. Read it before estimating an integration.

## What works today

- **Backend orchestration core**: tenant model, transaction persistence in Postgres (RLS-hardened), gasless EVM swap end-to-end (LiFi route + Biconomy MEE + EIP-7702 user-level delegation), Solana backend fee-payer co-signature path, transaction state machine.
- **The 6 endpoints flagged "Live"** in [API Reference](/docs/api-reference/introduction): `POST /v1/quote`, `POST /v1/prepare-tx`, `POST /v1/submit`, `POST /v1/execute-signed-quote`, `GET /v1/transactions/{id}`, plus tenant-scoped API-key auth.
- **API key management (M2)**: self-serve mint / list / revoke / rotate via [`/v1/api-keys`](/docs/api-reference/api-keys/create). `pk_*` and `sk_*` keys, hard-separated `test` / `live` environments, SHA-256 hashed at rest, replaces the M1 bearer token. CLI bootstrap (`npm run mint:key`) for issuing the first `sk_*` of a new tenant.
- **Webhook delivery (M3)**: register endpoints via [`/v1/webhook-endpoints`](/docs/api-reference/webhook-endpoints/create), HMAC-SHA256 signing with rotation overlap, exponential retry (7 attempts over ~67h), [replay + audit log](/docs/api-reference/webhook-events/list). `tx.prepared` / `tx.submitted` / `tx.failed` emit today; wallet / KYC events follow as their orchestration paths land.
- **Usage metering (M4)**: per-tenant monthly counters via [`/v1/usage`](/docs/api-reference/usage/current) + [`/v1/usage/history`](/docs/api-reference/usage/history). `active_users` computed at query time (always accurate); `wallets_created` + `tx_count` maintained as atomic UPSERT counters bumped on the originating events. `usd_volume` wiring deferred until LiFi USD price feed is plumbed through.
- **Backend deployed (M5.2 partial)**: live on AWS ECS Fargate (Express Mode) in eu-west-3. Single 0.25 vCPU / 0.5 GB Fargate task, HTTPS endpoint, all M2/M3/M4 surfaces reachable. The custom domain `api.fibo-crypto.fr` is the next step (ACM cert + DNS CNAME).
- **CLI (M5.1)**: `@fibo/cli` — `fibo keys|webhooks|events|usage <subcommand>` against any backend URL. Source-only today; public npm publication post-M5.
- **SDK v0.1 (M5.3)**: `@fibo/react-sdk` — `<FiboProvider />` + `useFiboAuth()` + `useFiboApi()` + typed `createFiboClient()` factory. Privy embedded wallets configured automatically; integrators don't install Privy directly (per ADR 0003). Distributed as a tarball today; public npm publication post-M5. Higher-level hooks (`useFiboSwap`, `useFiboBalance`, `useFiboWallet`, etc.) ship in 0.2+.
- **Reference React frontend**: an end-to-end working PoC web app that exercises the full quote → prepare → 2-signature → execute → confirmation flow. We share its code with onboarding integrators.
- **The full security and architecture story** — what's in the [Security](/docs/security) and [Architecture](/docs/concepts/architecture) pages is stable and reviewable today.

## What's "documented but not yet there"

These surfaces appear in the docs as production-shape because the contract is finalized — but the implementation isn't fully wired yet. Each carries a `:::info Pre-release` banner with the milestone it ships in:

| Surface | Today | Lands in |
|---|---|---|
| `@fibo/react-sdk` on npm | Distributed as a pinned tarball to the first cohort. Contact sales for access. | M5 |
| Dashboard UI (key rotation, webhook config, replay) | All admin operations are via API call or via the Fibo team. | M5 |
| `api.sandbox.fibo-crypto.fr` (dedicated sandbox base URL) | Sandbox runs against the same backend with `_test_` keys. | M5 |
| `wallet.created`, `tx.confirmed` and other long-tail events | Not yet emitted (M3 ships only `tx.prepared` / `tx.submitted` / `tx.failed`). Payload contracts published; handlers can be written now. | Phase 1 (wallet creation orchestration + on-chain watcher) |

## What's on the longer-term roadmap

- **React Native SDK** — same hook API as `@fibo/react-sdk`. Ships after M5.
- **SOC 2 Type II** — initial Type II report targeting Q4 2026.
- **Additional ramping connectors** — Stripe Onramp, Mercuryo, Mt Pelerin (each shipped as an opt-in `@fibo/react-sdk/<provider>` sub-export).
- **Cross-provider ramp routing** — pick the cheapest provider per user / amount / currency, automatically.
- **Additional chains** — Linea, zkSync Era, Mantle, Berachain on the radar; on-request only at this stage.

## How to interpret the existing docs

Most pages describe production-shape behavior — the contracts are finalized so you can write integration code against them now. Where a feature is in the "documented but not yet there" table above, the relevant page carries an `:::info Pre-release` banner that links back here.

If you're evaluating Fibo today, the trade-off is honest: **the architecture and security model are stable and reviewable now**, but you'll need to coordinate with us for credentials, webhook endpoints, and SDK access until M5 lands. Current integrators are comfortable with that — the white-glove path resolves any setup question within a business day.
