## Before you start

This quickstart assumes a **React frontend**. The Fibo SDK is React-only at MVP; native iOS / Android / Vue / Svelte support is on the roadmap. If your stack is something else, the REST API documented under [API Reference](/docs/api-reference/introduction) is fully usable today — you'll just have to wire the user-signing UI yourself.

You'll need:

- **Node.js ≥ 20.0** (`node --version` to check). Lower versions are not tested and several transitive deps emit engine warnings.
- **React 18 or 19** in your app.
- **A React framework** — Next.js (Pages or App router), Vite + React, Remix, Astro with `client:only="react"`, or plain React (CRA / similar). See [Frameworks](/docs/sdk/installation#frameworks).
- **A package manager** — npm, pnpm, or yarn. All three are tested.
- **A server endpoint** for receiving Fibo webhooks. HTTPS-only — Fibo refuses to deliver to `http://`. For prototyping, an ngrok / Cloudflare Tunnel URL works.
- **A Fibo account** — sign up at [dashboard.fibo-crypto.fr/onboarding/apply](https://dashboard.fibo-crypto.fr/onboarding/apply). Sandbox access is granted immediately on submission; you can complete steps 1-4 below without waiting for live-mode approval.

If you're starting from a blank Next.js app:

```bash
npx create-next-app@latest my-fibo-app --typescript --app
cd my-fibo-app
```

Skip this if you have an existing project.

## 1. Mint your API keys from the dashboard

Sign in at [dashboard.fibo-crypto.fr](https://dashboard.fibo-crypto.fr) → **API Keys** → **Mint a new key**. Two key types are issued per environment (`test` and `live`):

- **`pk_<env>_…`** — *publishable* key. Embedded in the browser bundle, safe to expose. Used by the SDK to authenticate users and to call browser-callable endpoints.
- **`sk_<env>_…`** — *secret* key. Server-side only. Used by your backend to call privileged endpoints (refunds, user listing, webhook secret rotation).

For this quickstart, mint **one `pk_test_*` and one `sk_test_*`** — both visible in plaintext only at the moment of creation (the dashboard shows them in an amber banner you can copy from).

```bash
# Put them in your .env.local — never commit
NEXT_PUBLIC_FIBO_PUBLISHABLE_KEY=pk_test_xxx
FIBO_SECRET_KEY=sk_test_xxx
```

:::note
**Never** ship `sk_…` to the browser. The SDK validates at construction time and throws if you try (`pk_*` only on the frontend). If you spot an `sk_*` in a frontend bundle, revoke and rotate from the dashboard immediately.
:::

## 2. Install the SDK

```bash
npm install @fibo/react-sdk --legacy-peer-deps
```

:::info Tarball install during PoC
Public npm publication ships post-M5. Today the SDK is distributed as a pinned tarball URL — paste it into your `package.json` as `"@fibo/react-sdk": "https://cdn.fibo-crypto.fr/sdk/react-sdk-0.x.y.tgz"`. The tarball URL is shown in your dashboard's SDK install page. Drop-in replacement when the public package ships.

The `--legacy-peer-deps` flag works around a Privy v2 peer-deps quirk; will go away in a future SDK release.
:::

The SDK has Privy as a transitive dependency — you don't need to install or configure Privy yourself.

## 3. Wrap your app

```tsx
import { FiboProvider } from '@fibo/react-sdk';

export default function App() {
  return (
    <FiboProvider
      publishableKey={process.env.NEXT_PUBLIC_FIBO_PUBLISHABLE_KEY!}
      // Optional: pass your own user identifier so Fibo can join its
      // record with your DB on the wallet.created webhook.
      externalUserId={currentUser.id}
    >
      <YourApp />
    </FiboProvider>
  );
}
```

    ```tsx
    import { useFiboSwap } from '@fibo/react-sdk';

    function SwapWidget() {
      const { quote, swap } = useFiboSwap();

      const handleSwap = async () => {
        // 1. Ask Fibo for a route + fee quote (returns within ~1s)
        const q = await quote({
          source:      { currency: 'USDC', chain_id: 8453, amount: '100' },
          destination: { currency: 'ETH',  chain_id: 8453 },
        });
        // 2. Execute it — Privy prompts the user to sign.
        await swap(q);
      };

      return <button onClick={handleSwap}>Swap 100 USDC → ETH</button>;
    }
    ```

    See [your first swap](/docs/guides/first-swap) for the full UX (quote breakdown, error states, confirmation).
  </TabItem>

      switch (req.body.event_type) {
        case 'wallet.created':  /* persist mapping */ break;

        case 'wallet.funded':   /* unlock features */ break;
      }

## What you have now

After completing the steps above:

1. Your users authenticate via the FiboProvider (email / OAuth / passkey, configurable).
2. On first login, Fibo provisions embedded wallets on the chains you opted into.
3. You receive a `wallet.created` webhook with the wallet addresses — store these against your user record.
4. The SDK exposes hooks for balances, swaps, transfers, and yield operations. Every state-changing action returns a Promise that resolves once the user has signed.
